# The bin/ toolbelt

Firstmate drives these; interactive entrypoints work by hand too, while `*-lib.sh` files are sourced helpers.
Each file also starts with a short header comment.
If you have changed away from the firstmate home in an interactive shell, invoke these scripts by absolute path through the repo's `bin/` directory; the scripts self-locate internally after they start.

Scripts are grouped by their disposition under the dynamic-workflow paradigm (AGENTS.md, disposition table in the paradigm PR).
Nothing is deleted here: retired and escape-hatch scripts stay functional on disk until the cleanup PR, which is held until after the Opus 4.8 transition settles, so rollback stays real the whole time.

- **New** - built for the workflow paradigm.
- **Kept** - active in the workflow paradigm.
- **Escape hatch** - deprecated for new dispatch, functional on disk, used only on David's explicit request (the tmux-crewmate stack for non-Claude harnesses or must-outlive-session work, and the local-only delivery mode).
- **Retired** - superseded; each carries a `DEPRECATED` header, removed in the cleanup PR.
- **Inert** - X mode, off until opted in.

| `fm-gfetch.sh`        | Deterministic read-only Gmail/Drive/Calendar fetch wrapper for the meeting-sync Stage A FETCH (meeting-sync-design.md Decision 4b, Phase 0b), the Google analogue of `fm-linear.sh`: drives Google's own REST APIs (Gmail v1, Drive v3, Calendar v3) directly with a durable OAuth access token, so the reads behave identically from a subagent brief, a cron run, or an interactive shell with no per-session MCP-loading dependency. Subcommands: `status` (credential probe, presence + source only, never the token), `threads`/`thread` (search + read the "Kronos Tech Sync" Gemini-notes threads), `files`/`doc` (find + read a Drive doc's structured notes + transcript as plain text), `events`/`event` (read a calendar event with a computed `morning`/`eod` `slot_hint` that applies the CST->PT trap and Eric/Yujie attendee discriminator from `data/meetings-cadence.md`); `--raw` prints the unmodified Google JSON. Credential resolution mirrors `fm-linear.sh`, first hit wins and never printed: `$FM_GOOGLE_ACCESS_TOKEN`, then `$FM_GOOGLE_CRED_FILE` (default `~/.config/fm-gfetch/credentials.json`, minting a fresh token from a `refresh_token` shape), then `gcloud`, then the Keychain/`~/.claude/.credentials.json` mcpOAuth store. Per design open question 10 no durable Google credential exists yet, so the load-bearing behavior is the honest degrade (Decision 4b Option C): an absent/expired/rejected credential makes a fetch print one machine-parseable `notes-not-fetchable ... paste-them` line to stdout, a recovery hint to stderr, and exit 3 - never hangs, never leaks a token. Exit codes: 0 ok, 2 usage, 3 credential unavailable/degrade, 4 Google API error, 5 network. No launchd plist: a one-shot CLI whose schedule is cron-owned (Decision 7a), not the poller.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |

The stall detector `state/workflow-runs.check.sh` (journal mtime vs phase budget) is a runtime-generated check under the gitignored `state/`, not a committed script; the poller runs it like any other `*.check.sh`.
