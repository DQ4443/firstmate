# The bin/ toolbelt

Firstmate drives these; interactive entrypoints work by hand too, while `*-lib.sh` files are sourced helpers.
Each file also starts with a short header comment.
If you have changed away from the firstmate home in an interactive shell, invoke these scripts by absolute path through the repo's `bin/` directory; the scripts self-locate internally after they start.

Scripts are grouped by their disposition under the dynamic-workflow paradigm (AGENTS.md, disposition table in the paradigm PR).
Nothing is deleted here: retired and escape-hatch scripts stay functional on disk until the cleanup PR, which is held until after the Opus 4.8 transition settles, so rollback stays real the whole time.

- **New** - built for the workflow paradigm.
- **Kept** - active in the workflow paradigm.
- **Escape hatch** - deprecated for new dispatch, functional on disk, used only on David's explicit request (the tmux-crewmate stack for non-Claude harnesses or must-outlive-session work, and the local-only delivery mode).
- **Retired** - superseded; each carries a `DEPRECATED` header, removed in the cleanup PR.
- **Inert** - X mode, off until opted in.

## New (workflow paradigm)

| Script                | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `fm-poll.sh`          | Launchd-supervised board/merge/stall poller (`com.firstmate.poller`, KeepAlive). Loops every `state/*.check.sh` on the check contract (print one line only on a wake-worthy event), delivers each event through the durable wake queue in `fm-wake-lib.sh`, maintains its own `state/.last-poller-beat` liveness beacon, holds a home-scoped singleton pid guard with a process-identity check (so a recycled pid cannot wedge it in a silent respawn loop), and injects a rate-limited synthetic startup check event so a wake confirms the delivery path end to end. Replaces `fm-watch.sh` for daily duty; not armed by hand.                                                                                                                                                                                                                                                                                                                                         |
| `fm-write-fence.sh`   | Broadened PreToolUse write fence (Edit/Write/NotebookEdit). Resolves the target's real path (walking up to an existing ancestor, so a new file created through a symlink still resolves into the real tree) and blocks writes under `~/dev/work/**` and under `<FM_ROOT>/projects/**` (firstmate's own clones, which are real dirs, not symlinks into `~/dev/work`) except the isolated worktrees `~/dev/work/*/.claude/worktrees/**`, `projects/*/.claude/worktrees/**`, and `~/.treehouse/**`; fails open on unreadable payloads. Best-effort structural enforcement of AGENTS.md prime rules 2 and 3 (defense-in-depth behind isolation), broadened from the old `*.html`-only `block-html-edits.sh`. Wired locally at the poller cutover, not in tracked settings.                                                                                                                                                                                                   |
| `fm-board-checkin.sh` | Board check-in writer. Sets `state/board-checkins.json[<item-id>]` to a real epoch (default now), merging into the existing object and writing atomically, so the board's "last checked Xm ago" stamp is a fact. Called deterministically by an orchestration script at every phase boundary (AGENTS.md section 4).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `fm-ledger.sh`        | Resumable loop-ledger for a long autonomous run: `create`/`update`/`read` one JSON per run under `state/ledgers/<run>.json` carrying objective, phase, appended decisions, the next contract-critical action, and status. The mechanical carrier for run state that a late step depends on (AGENTS.md section 4), so a resuming or post-compaction firstmate recovers where a run is from disk rather than a compacted-out window. Requires `jq`, rejects path-escaping run ids, and writes atomically (never clobbers a live ledger on a render failure).                                                                                                                                                                                                                                                                                                                                                                                                               |
| `fm-sync-audit.sh`    | Append-only per-write audit log for the meeting-sync loop (Phase 0 substrate, meeting-sync-design.md section 9a). `append <slot> <target> <op>` writes one JSON line per write to `data/meeting-sync-audit/<slot-id>.jsonl` (timestamp, slot/run id, target ENG id or tracker node/item ref, op, before/after, transcript-timecode evidence) as a single O_APPEND write + fsync the moment the op lands, so a crash leaves a correct partial record; `read [--newest-first] [--target T] [--op O]` returns a JSON array in file/append order (newest-first is the order the reverse-run rollback in section 9b consumes, never a coarse timestamp), and `slots`/`path` enumerate and locate logs. Refuses (exit 3) any value that matches a sensitive env var's value or carries a `Bearer` header, and rejects path-escaping slot ids before any write. Read/append-only local provenance under untracked `data/`; no Linear/tracker/narrative writes and no scheduler. |

Companion artifacts outside `bin/`: `launchd/com.firstmate.poller.plist` (the LaunchAgent for `fm-poll.sh`; ships unloaded, with a header comment - loading it is the human-verified cutover step, and the placeholders `__FM_ROOT__`/`__FM_HOME__` are substituted then).
The stall detector `state/workflow-runs.check.sh` (journal mtime vs phase budget) is a runtime-generated check under the gitignored `state/`, not a committed script; the poller runs it like any other `*.check.sh`.
