# Evals for `$rig-atlas`

Run isolated graders with binary checks.
The target is at least 90 percent across three runs.

## Should trigger

1. A spine skill or harness hook changed and the rig documentation must be refreshed.
2. Codex role definitions or default routing changed.
3. The current workflow setup must be inventoried and reflected in the existing system study.
4. A durable house rule changed inside a live rig skill.
5. A periodic drift check is due.

## Should not trigger

1. Product or repository architecture documentation.
2. A wording-only edit inside one skill when no atlas refresh was requested.
3. A question asking for the contents of one memory file.

## Binary checks

- [ ] The current atlas, source inventory, and generator were read before the live inventory began.
- [ ] Five separate scout-shaped PDWs covered skills, harness, roles, conventions, and memory in parallel.
- [ ] Read-cell counts were selected dynamically for each surface.
- [ ] One convergence PDW consumed only funneled surface briefs.
- [ ] Funnels rejected degenerate output, verified paths, named dead lanes `UNVERIFIED`, and deduplicated globally.
- [ ] The run produced a concrete drift list before edits.
- [ ] The source audit reconciled exactly 47 include-both, 47 full-only, and 41 exclude-both classifications.
- [ ] Appendix D contained exactly the 47 include-both bodies and no full-only or excluded body.
- [ ] The generator inventory retained nine spine skills and three role definitions.
- [ ] Domain skills appeared only in the live inventory.
- [ ] Exactly one generated full atlas existed at `state/rig/rig-atlas.md`.
- [ ] The full atlas was changed only through the generator.
- [ ] The portable twin did not exist while the sanitizer remained redacted.
- [ ] Portable generation failed nonzero without writing an output.
- [ ] No full-only or excluded memory body was copied into tracked or runtime output.
- [ ] Runtime output under `state/rig/` remained ignored and uncommitted.
- [ ] Optional harness surfaces were claimed only after an existence check.
- [ ] State conventions were documented without live task contents.
- [ ] The existing system-study surface was updated in place after generation when the host provides it.
- [ ] The return carried `NEXT_STEP: regenerate the full rig atlas, then update the existing system-study surface last`.
